## 1. 项目背景与目标概述

### 1.1 项目背景

本项目是NIS3366《项目管理与软件设计》课程设计一的项目，旨在使用HarmonyOS ArkTS开发一个基于HarmonyOS的商城应用。随着移动互联网的快速发展，电商应用已成为人们日常购物的主要渠道。HarmonyOS作为华为自主研发的操作系统，其分布式特性为移动电商应用提供了新的可能性。

本项目以"毕办商城"为产品名称，面向校园用户群体，提供基础的商品浏览、购物车管理、个人中心等核心电商功能。项目采用本地模拟数据的方式实现，便于课程演示与功能验证。

### 1.2 项目目标

本项目的核心目标包括：

1. **功能完整性**：实现电商应用的核心购物流程，包括商品浏览、分类筛选、购物车管理、订单结算等
2. **用户体验优化**：提供流畅的交互体验，包括下拉刷新、上拉加载、列表懒加载等性能优化
3. **架构合理性**：采用清晰的分层架构（页面层、服务层、数据层），便于后续功能扩展
4. **技术实践**：熟练掌握HarmonyOS ArkTS语言和ArkUI声明式UI框架

### 1.3 业务场景

本应用主要服务于以下业务场景：

| 场景 | 描述 | 目标用户 |
|------|------|----------|
| 商品浏览 | 用户在首页浏览商品，可通过分类筛选快速定位目标商品 | 所有用户 |
| 商品详情 | 查看商品详细信息（名称、价格、图片、分类） | 有购买意向的用户 |
| 购物车管理 | 将商品加入购物车，修改数量，删除商品 | 已确定购买商品的用户 |
| 订单结算 | 选择购物车中的商品进行结算 | 确认购买的用户 |
| 个人中心 | 查看用户信息、订单状态、常用功能入口 | 已登录用户 |

## 4. 技术要求

### 4.1 开发环境与工具

本项目开发环境要求如下：

| 项目 | 要求 |
|------|------|
| 目标系统 | HarmonyOS Next |
| 开发语言 | ArkTS |
| UI框架 | ArkUI声明式UI框架 |
| 开发工具 | DevEco Studio |
| 部署环境 | 鸿蒙真机设备或模拟器 |

### 4.2 技术栈要求

本项目技术栈要求包括：

1. **开发语言**：必须使用ArkTS语言进行全部开发工作
2. **UI框架**：使用ArkUI声明式UI框架进行界面开发
3. **组件要求**：使用Scroll组件、List组件以及LazyForEach组件实现商品列表页面
4. **数据加载**：利用LazyForEach实现列表懒加载，优化性能

### 4.3 开发阶段要求

本项目开发过程应经历以下五个阶段：

1. **分析设计阶段**：完成需求分析，明确功能点和业务流程
2. **绘制商品页面阶段**：使用ArkUI组件绘制商品展示页面
3. **加载页面阶段**：实现商品数据的加载与展示
4. **支持列表滑动阶段**：优化列表滑动性能，实现流畅的滚动体验
5. **支持更多品类阶段**：扩展商品分类，完善筛选功能

### 4.4 功能实现要求

功能实现需满足以下要求：

1. **下拉刷新**：商品列表支持下拉刷新功能，更新最新数据
2. **懒加载**：使用LazyForEach实现列表数据的懒加载，提升首屏加载速度
3. **到底提示**：列表滚动到底部时显示提示信息，提示用户已加载全部数据

## 5. 系统功能需求清单

### 5.1 功能优先级划分说明

本需求文档将功能需求划分为三个优先级：

| 优先级 | 定义 | 说明 |
|--------|------|------|
| P0（核心功能） | 必须实现 | 构成电商应用基础购物流程的必备功能 |
| P1（重要功能） | 应该实现 | 提升用户体验和完善业务流程的重要功能 |
| P2（辅助功能） | 可以实现 | 增强产品竞争力的扩展功能 |

### 5.2 核心功能需求（P0）

#### 5.2.1 商品浏览功能

| 需求项 | 详情 |
|--------|------|
| **功能描述** | 用户在首页能够浏览商品列表，以网格布局展示每个商品的名称、价格和图片信息 |
| **业务规则** | 1. 商品列表采用双列网格布局<br>2. 使用LazyForEach实现列表懒加载<br>3. 支持下拉刷新功能<br>4. 滚动到底部时显示提示 |
| **输入输出** | 输入：无<br>输出：商品列表数据（包含商品ID、名称、价格、图片、分类） |
| **异常处理** | 1. 数据加载失败：显示错误提示，提供重试按钮<br>2. 无商品数据：显示空状态提示 |
| **验收标准** | 1. 首页能够正常加载并显示商品列表<br>2. 商品图片、名称、价格正确显示<br>3. 下拉刷新能够触发数据重新加载<br>4. 滚动流畅，无明显卡顿 |

#### 5.2.2 分类筛选功能

| 需求项 | 详情 |
|--------|------|
| **功能描述** | 用户能够通过顶部分类导航筛选特定品类的商品 |
| **业务规则** | 1. 分类导航横向滚动显示<br>2. 支持的分类包括：全部、电子产品、服装鞋帽、家居用品、美妆护肤、食品生鲜<br>3. 选中分类高亮显示（红色背景）<br>4. 切换分类后自动滚动到列表顶部 |
| **输入输出** | 输入：选中的分类ID<br>输出：筛选后的商品列表 |
| **异常处理** | 1. 分类数据为空：显示默认"全部"分类<br>2. 分类切换失败：保留当前显示内容 |
| **验收标准** | 1. 分类导航能够横向滚动<br>2. 点击分类能够正确筛选商品<br>3. 选中状态视觉反馈正确<br>4. 切换分类后列表自动滚动到顶部 |

#### 5.2.3 购物车添加功能

| 需求项 | 详情 |
|--------|------|
| **功能描述** | 用户能够在商品列表页将商品添加到购物车 |
| **业务规则** | 1. 点击商品卡片上的"加入购物车"按钮添加<br>2. 如果商品已存在于购物车中，则数量+1<br>3. 添加成功后显示成功提示<br>4. 购物车数量角标实时更新 |
| **输入输出** | 输入：商品ID<br>输出：购物车状态更新结果 |
| **异常处理** | 1. 添加失败：显示错误提示<br>2. 商品不存在：显示商品不存在提示 |
| **验收标准** | 1. 点击"加入购物车"按钮能够添加商品<br>2. 重复添加同一商品，数量正确累加<br>3. 添加成功有视觉/Toast反馈<br>4. 底部导航栏购物车图标角标数量正确 |

#### 5.2.4 购物车查看功能

| 需求项 | 详情 |
|--------|------|
| **功能描述** | 用户能够查看购物车中已添加的所有商品 |
| **业务规则** | 1. 购物车页面展示商品列表<br>2. 每个商品项显示：商品图片、名称、价格、数量<br>3. 购物车为空时显示空状态提示和"去逛逛"按钮<br>4. 支持下拉刷新 |
| **输入输出** | 输入：无<br>输出：购物车商品列表数据 |
| **异常处理** | 1. 购物车为空：显示空状态页面<br>2. 数据加载失败：显示错误提示 |
| **验收标准** | 1. 购物车页面正确显示所有已添加商品<br>2. 空购物车显示空状态提示<br>3. 点击"去逛逛"能够跳转到首页<br>4. 下拉刷新能够重新加载数据 |

#### 5.2.5 购物车数量修改功能

| 需求项 | 详情 |
|--------|------|
| **功能描述** | 用户能够增加或减少购物车中商品的数量 |
| **业务规则** | 1. 每个商品项显示"+"和"-"按钮<br>2. 点击"+"按钮数量+1<br>3. 点击"-"按钮数量-1<br>4. 当数量减少到0时，自动从购物车中移除该商品 |
| **输入输出** | 输入：商品ID、操作类型（增加/减少）<br>输出：更新后的购物车状态 |
| **异常处理** | 1. 数量修改失败：显示错误提示，保持原数量<br>2. 数量为0时自动移除商品 |
| **验收标准** | 1. 点击"+"按钮数量正确增加<br>2. 点击"-"按钮数量正确减少<br>3. 数量为0时商品自动移除<br>4. 总价实时更新 |

#### 5.2.6 购物车删除功能

| 需求项 | 详情 |
|--------|------|
